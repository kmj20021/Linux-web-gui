## Root Cause Analysis

### Issue 1: Admin Router Not Loaded (Critical)
- **Component:** Backend
- **File:** /home/ubuntu/Linux-web-gui/backend/routers/admin.py
- **Cause:** ImportError during server startup
- **Evidence:** 
  - File exists (/home/ubuntu/Linux-web-gui/backend/routers/admin.py, 8964 bytes, dated 2026-05-17 08:13)
  - Imported in main.py correctly
  - Registered in main.py correctly
  - NOT present in OpenAPI paths
  - All other routers (auth, monitor, network, shell) load successfully
- **Impact:** Cannot test:
  - GET /api/admin/users
  - POST /api/admin/users
  - PATCH /api/admin/users/{id}
  - DELETE /api/admin/users/{id}
  - GET /api/admin/audit

### Issue 2: Register Endpoint Returns Wrong Role (Critical)
- **Component:** Backend
- **File:** /home/ubuntu/Linux-web-gui/backend/routers/auth.py
- **Cause:** Line 208 sets `role="admin"` but WebUser model default is "viewer" (models.py:99)
- **Problem:** When WebUser is created, the role parameter is being ignored or overridden
- **Evidence:** 
  - POST /api/auth/register with any user returns role="viewer"
  - POST /api/auth/login for same user returns role="viewer"
  - Default admin account created by ensure_default_admin() correctly gets role="admin"
- **Impact:** No newly registered users can become admin - all start as viewer
